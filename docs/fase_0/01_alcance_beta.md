# 01 — Alcance de la Beta

## Qué incluye la beta
La beta inicial debe enfocarse estrictamente en el trabajo operativo diario de Torre de Control. Los módulos mínimos incluidos son:
1. Reporte diario (selección de fecha).
2. Brigadas del día (registro y control de brigadas).
3. Programación por zona (ingreso de datos manuales).
4. Parámetros básicos CYR.
5. Resumen automático por zona.
6. Exportación a Excel.

## Qué NO incluye la beta
No se deben agregar módulos avanzados en esta etapa. Quedan excluidos para fases posteriores:
- Usuarios por rol y Login avanzado.
- Módulo para supervisores.
- Módulo para brigadas (App móvil).
- Dashboard gerencial avanzado.
- Bonificación compleja.
- Inteligencia Artificial visible dentro de la app.
- Notificaciones automáticas o push.
- Auditoría avanzada de cambios.

## Por qué se excluyen módulos avanzados
El objetivo de la beta es crear un reemplazo rápido, seguro y funcional para el Excel actual. Añadir roles, dashboards o integración móvil en este momento aumentaría el tiempo de desarrollo y la complejidad operativa sin resolver el problema inmediato de Torre de Control: la gestión diaria de datos y reportes.
